    You will get all the service items that are included in the plan, this services are ideal to explain to the user which features are included in he plan. If the "coupons" key on the response payload has a none-empty array, it meanse this plan has coupons that need to be "auto aplied" If the coupon is applied, we need to show the real price crossed over and the new price with the discount applied.

    2) Then the user needs to fill the card and press the button to pay:

    We call the first endpoint first to save the card on the payment platform (stripe)

    POST /v2/payments/card Request Payload: 

    {"card_number":"4242424242424242","exp_month":"10","exp_year":"26","cvc":"435","academy":47}

    Response payload like this: Status: 200 

    {
    "status": "ok",
    "details": {
        "4Geeks.com": {
            "card_last4": "4242",
            "card_brand": "Visa",
            "card_exp_month": 10,
            "card_exp_year": 2026
        }
    }
}
